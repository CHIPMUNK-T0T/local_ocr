# Local OCR Project

このプロジェクトは、ローカル環境で各種 OCR モデルを比較・検証するための作業領域です。

## セットアップ情報

- 仮想環境名: `ocr_venv`
- Python バージョン: `3.12.3`
- 主要ライブラリ: `docling`, `transformers`, 各 OCR モデル実装

## 仮想環境の使用方法

以下のコマンドで仮想環境を有効化できます。

```bash
source ocr_venv/bin/activate
```

## 開発ルール

- 依存関係を追加・更新した場合は、`requirements.txt` など再現に必要なファイルへ反映する。
- OCR の実行確認は原則 GPU 前提で行う。
- GPU が使えない場合は、CPU に安易にフォールバックせず、先に原因を報告する。
- モデルごとの依存競合を避けるため、OCR モデルは仮想環境を分離して管理する。
- 現在動いている環境はそのまま活用し、動いていないモデルだけを新しい仮想環境へ移行する。
- 各モデルクラスは、自分に割り当てられた仮想環境の Python を使って worker を呼び出せるようにする。
- OCR モデル実装は共通の抽象クラス契約に厳密に従う。
- 各モデルクラスは同じ責務と同じメソッド群を持ち、ランナー側にモデル固有分岐を持ち込まない。
- 実装が複雑になりすぎる場合は、モデル固有処理と共通処理を分離し、共通部分は base / utils に寄せる。
- 実装の詳細は `GEMINI.md` の内容も参照する。

## 仮想環境ポリシー

- `docling`: 現在は `ocr_venv`
- `glm`: 現在は `ocr_venv`
- `dots`: `venvs/dots_vllm_venv`
- `paddle`: `venvs/paddle_venv`
